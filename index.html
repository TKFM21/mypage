  12. js:thisの使い方：4パターン
    1. メソッド呼び出しパターン

      オブジェクトのメソッドを呼び出した場合、thisはそのオブジェクトが対象となる

      
                        var myObject = {
                            val: 10;
                            show: function(){
                                console.log(this.val);
                                console.log(this);
                            }
                        }
                        myObject.show(); //10が出力される
    2. 関数呼び出しパターン

      ここでのthisはwindowオブジェクトが対象

      
                        function show(){
                            console.log(this);
                            this.val = 1;
                        }
                        show(); //window.show()オブジェクトが出力される

      オブジェクトのメソッドで関数を作成した場合もwindowオブジェクトが対象となる

      
                        var myObject = {
                            val: 1;
                            show: function(){
                                console.log(this.val); //1
                                console.log(this); //myObjectが対象

                                function myFunc(){
                                    console.log(this.val); //undefined
                                    console.log(this); //windowオブジェクトが対象のため
                                }
                                myFunc();
                            }
                        };
                        myObject.show();
    3. コンストラクタ呼び出しパターン

      オブジェクトインスタンスが対象

      
                        function MyObject(val){
                            this.myVal = val;
                            this.myFunc = function(){
                                this.myVal++;
                            };
                        }
                        var myObject = new MyObject(1); //ここで作成されるインスタンスがthisの対象
                        console.log(myObject.myVal); //出力は1
                        myObject.myFunc();
                        console.log(myObject.myVal); //出力は2
    4. apply,call呼び出しパターン

      thisの対象を切り替えることができる

      
                        var myObject = {
                            val: 1;
                            show: function(){
                                console.log(this.val);
                            }
                        };
                        var yourObject = {
                            val: 3;
                        };

                        myObject.show(); //1
                        myObject.show.apply(yourObject); //3
                        myObject.show.call(yourObject); //3

  1. js:クロージャ

    PHPなどとは異なる仕様

    「自分を囲むスコープにある変数を参照できる関数」がクロージャ

    関数の実行が完了されればCallオブジェクトは破棄されるが、さらに内側などでローカル変数が利用されていれば保持される

    
                    //開発ツールのコンソールへ直接コーディングしても動作を確認できないため、実ファイルで確認した
                    function outerFunc(){
                        var value = 1;

                        //outerFunc Callオブジェクト ={
                        //  arguments : ....
                        //  this : ....
                        //  value : 1
                        //}
                        
                        function innerFunc(){
                            //innerFunc Callオブジェクト = {
                            //  arguments : ....
                            //  this : ....
                            //}
                            console.log(++value);
                        }
                        return innerFunc;
                    }
                    var fn1 = outerFunc(); //innerFuncという関数オブジェクトが変数に格納されている
                    fn1(); //出力は2
                    fn1(); //出力は3
                    //innerFuncのCallオブジェクトはouterFuncのCallオブジェクトのローカル変数を参照しているので破棄されず残ったままになる
  2. クロージャを使ってプライベート変数が作成できる！ 
    
                    //開発ツール上では動作を確認できないため、ファイルで確認した
                    <script type="text/javascript">
                        var Module = function(){
                            var count = 0;
                            return {
                                increment: function(){
                                    alert(count++);
                                }
                            };
                        };
                        var md1 = new Module();
                        alert(md1.count); //undefined
                        md1.increment(); //0
                        md1.increment(); //1
                        alert(md1.count); //undefined
                    </script>
  3. クロージャの使い所
    • グローバル変数の宣言をなるべく減らしたいとき
    • ユーザーが引数を与えてカスタマイズ可能な自由度の高い「関数」を生成したいとき
    • 前回、呼び出されて実行されたときの演算結果（値）を内部で保持して、次に呼び出されたときに、前回の結果（値）に対して、更に同じ処理を行う関数を生成したいとき
  4. クロージャ課題：ボタンカウンター：[js_button_counter]で作成

  3. javascriptでcookieを編集：cookieの概要
    • データをそのままブラウザに保存しておける機能
    • ブラウザごとに保存されるので、別のブラウザで同じcookieの中身は見られない。（データ共有不能）
    • ブラウザに保存されているため、ユーザーが改変できる。
    • そのため個人情報は保存してはいけない
    • ブラウザで保存できるcookieの個数とバイト数はブラウザによって微妙に異なる
    • IEでは、1ドメインにつき50個のcookieが保存でき、1個のcookieは4096バイトまで
    • cookieの名前や値には[;][,]空白を含めてはいけない（encodeURIComponent()を使用して排除）
    • cookieの中にはkey、valueの形式でデータが保存されている
    • chromeにはcookie管理用に拡張機能EditThisCookieを追加すること
    • jquery-cookie-masterを使うことでコーディングが短縮できる
    • サンプルコードは「js_cookie」へ

  2. オンライン教室：jsの非同期処理promiseとasync/await
    • promiseの学習の流れ：白熱教室で概要を掴む　ｰ>　ここで詳細を理解
    • promiseはクラス
    • promiseには3つの状態
      1. pending:処理中（インスタンスが作成されてresolveかrejectが呼ばれる前の状態）
      2. resolve:正常完了
      3. reject:異常終了
    • promiseは引数に2つのコールバック関数が指定可能。一般的には(resolve, reject)のような書き方
    • thenメソッドとcatchメソッドはセットで指定するのが一般的
      1. thenメソッドはインスタンスでresolveが呼び出されたときに実行される
      2. catchメソッドはエラー発生して、rejectが呼び出されたときに実行される
      3. エラーが起こればthenは無視、エラーがなければcatchは無視
      4. resolve(),reject()で（）に値を入れなかった場合、暗黙的にpromiseインスタンスが返される
      5. then,catchで何もreturnしなかった場合も暗黙的にpromiseインスタンスが返る
      6. 故にチェーンでthen,catchメソッドが繋げられる
      7. resolve(1)としてthen(value)とするとvalueに1が代入される仕組み
      8. catchの場合はcatch(error)でエラーを受け取って処理する
      9. thenメソッドでreturnでnew Promiseして非同期処理を追加することも可能
    • fetch(url)との組み合わせも重要
      1. fetchの内部処理ではpromiseインスタンスが作成されていて、かつ、resolve/rejectが実行される
      2. fetch(url).then((response) =>{responseに対する処理})
    • async, awaitはPromiseインスタンス作成後のthenメソッドの書き方を同期処理のようにわかりやすくしてくれる
